
# Техническая спецификация проекта: «Исторические даты»
## 1. Стек технологий и распределение ролей
Для реализации проекта необходимо использовать следующие инструменты:
 * **React + TypeScript (ReactTS):** Основа приложения. TypeScript обеспечивает типизацию данных (периодов и событий), а React управляет состоянием активного периода.
 * **Webpack:** Используется для сборки проекта «с нуля». Необходимо настроить загрузчики для TypeScript, SCSS и статических файлов (изображений/шрифтов).
 * **SASS/SCSS:** Для написания стилей. Рекомендуется использовать CSS-модули или методологию БЭМ для обеспечения независимости блоков.
 * **GSAP:** Главный инструмент для реализации всех сложных анимаций: вращения круга, перемещения точек и эффекта «бегущих цифр» в заголовках лет.
 * **Swiper:** Используется исключительно для нижнего слайдера событий. Позволяет реализовать навигацию стрелками и поддержку touch-событий.
## 2. Подробное описание дизайна
### 2.1. Версия для ПК (Desktop)
 * **Сетка и фон:** На фоне присутствуют тонкие декоративные линии (горизонтальная и вертикальная), пересекающиеся в центре. Это часть верстки, а не системная сетка.
 * **Центральный элемент:** Окружность диаметром около 530px. На ней расположены интерактивные точки.
 * **Интерактивные точки:**
   * **Активная:** Увеличена, имеет белый фон и порядковый номер. Рядом отображается название категории (категория появляется плавно).
   * **Неактивная:** Маленькая точка. При наведении (hover) увеличивается и показывает номер.
 * **Годы:** Два крупных числа (Blue: #42567A, Pink: #EF5DA8) в центре круга.
 * **Навигация:** Кнопки переключения (влево/вправо) и дробный счетчик (например, 01/06) находятся в левой части блока.
### 2.2. Мобильная версия (Mobile)
 * **Упрощение:** Круговая навигация полностью отсутствует.
 * **Компоновка:** Заголовок и годы остаются сверху. Название категории перемещается под годы и отделяется горизонтальной линией.
 * **Управление:** Вместо точек на круге используются стандартные буллиты (пагинация) Swiper под слайдером.
 * **Слайдер:** Занимает большую часть экрана, навигационные стрелки скрыты (используются жесты).
## 3. Логика анимаций (GSAP)
### 3.1. Вращение круга
При клике на точку или кнопку навигации:
 * Рассчитывается кратчайший угол поворота круга, чтобы выбранная точка оказалась в позиции top-right (примерно 30 градусов от вертикали).
 * **GSAP Rotation:** Вращается основной контейнер круга.
 * **Анти-вращение точек:** Чтобы цифры внутри точек не перевертывались, к ним применяется анимация вращения в противоположную сторону на тот же угол.
### 3.2. Эффект «Бегущие цифры»
При смене периода годы плавно меняются:
 * Используется объект-прокси в GSAP, который меняет числовое значение от oldYear до newYear.
 * Функция onUpdate обновляет текст в DOM-узле на каждом шаге анимации.
### 3.3. Смена контента (Слайдер)
 * При переключении периода текущий слайдер событий должен исчезнуть (opacity: 0, y: 20).
 * После завершения анимации круга новый контент появляется с эффектом fade-in и возвращается в исходную позицию по оси Y.

## 5. Инструкции по реализации
 1. **Математика круга:** Для позиционирования точек используйте формулу:
   * x = cx + r * cos(angle)
   * y = cy + r * sin(angle)
   * Угол для каждой точки: (360 / общее_кол-во) * индекс.
 2. **Независимость:** Весь код должен находиться внутри одного корневого компонента. Избегайте использования глобальных селекторов в SCSS, чтобы при добавлении второго блока на страницу анимации не конфликтовали.
 3. **Ограничения:** Не используйте UI-библиотеки (MUI, AntDesign) или CSS-фреймворки. Все декоративные элементы и кнопки должны быть сверстаны вручную.
